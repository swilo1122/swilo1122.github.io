总结一些SQL语句

By 马占臣

發表於 2015-09-02

无论是使用SQLite3还是FMDB来进行数据存储，我们都需要使用SQL语句来对数据句进行操作。数据库是一门很深奥的技术，我们做移动端开发的暂时不往深了探究，总结一些我们平时用的比较多的SQL语句吧。

简单地说SQLite数据库对应的就是一个文件（一般命名为xxx.db）。其中数据库的内部结构是以表（table）的形式呈现的。表是以行和列的形式组织起来的数据的集合。一个数据库包括一个或多个表。例如，可能有一个有关学生信息的名为t_student的表。每列都包含特定类型的字段，如姓名。每行都包含有关学生的所有信息：姓名、性别，分数等等。在关系型数据库当中一个表就是一个关系，一个关系数据库可以包含多个表。列又称为属性或字段（column），行又称为记录（row，record）。

SQL语句种类
DDL（数据定义）语句：创建表create table， 删除表drop table
DML（数据操作）语句：插入insert， 删除delete， 修改update
DQL（数据查询）语句：查询select， 条件：where、 order by、group by、having（这些关键字后面具体介绍）

数据库SQL语句中不分大小写，但是我们习惯将关键字大写，我们定义的表明字段等小写，并且表名多以t_开头，语句以;结束。

想要使用数据库，我们首先要建立一个表：

CREATE TABLE IF NOT EXISTS 表名 (字段1 类型, 字段2 类型);

此处注意IF NOT EXISTS关键字，加上这一行如果表名重复了不会出错，即：如果没有这个表，则创建，有这个表则什么都不做也不会出错。
一般表的结构在建表时要考虑清楚，后期修改相对麻烦一些。
删除表同理：

DROP TABLE IF EXISTS 表名;

修改表结构

增加一个列：
ALTER TABLE 表名 ADD(列名 数据类型);
修改一个列的数据类型(一般限于修改长度，修改为一个不同类型时有诸多限制):
ALTER TABLE 表名 MODIFY(列名 数据类型);
修改列名：
ALTER TABLE 表名 RENAME COLUMN 当前列名 TO 新列名;
删除一个列：
ALTER TABLE 表名 DROP COLUMN 列名;
将一个表改名：
ALTER TABLE 当前表名 RENAME TO 新表名;

向表中添加数据：

INSERT INTO 表名(字段1, 字段2) VALUES (‘字段1的值’, ‘字段2的值’) —（如果是字符串需要加’’）;

修改表中数据：

UPDATE 表名 SET 字段1 = 字段1的值, 字段2 = 字段2的值。。。;

查询：

SELECT 字段名, 字段名 FROM 表名;

删除表中的数据：

DELETE FROM 表名;

其中查询或修改或删除表中的数据一般都带有条件：

WHERE 字段名 条件 AND/OR 字段名 条件;
ORDER BY 字段名 DESC（降序，默认为升序）;
“Group By”从字面意义上理解就是根据“By”指定的规则对数据进行分组，所谓的分组就是将一个“数据集”划分成若干个“小区域”，然后针对若干个“小区域”进行数据处理。
GROUP BY 字段名;
HAVING 和WHERE都是用来筛选用的,HAVING 是筛选组,而WHERE是筛选记录.当分组筛选的时候 用HAVING,其它情况用WHERE;
用HAVING就一定要和GROUP BY连用，用GROUP BY不一用HAVING（它只是一个筛选条件用的）
HAVING 字段名 条件;

LIMIT可以精确得控制查询结果的数量，LIMIT一般放在最后面
LIMIT 数值1, 数值2;
上条语句的意义为跳过数值1条数据，然后取数值2条数据
LIMIT 10 相当于 LIMIT 0, 10

正常我们在创建表的时候还需要对字段添加一些约束来使数据库更容易修改和访问，也方便我们程序员之间的互相交流。
简单约束：
NOT NULL 表示字段的值不能为空
UNIQUE 规定字段的值必须唯一
DEFAULT 指定字段的默认值
PRIMARY KEY 主键（主键约束默认带有UNIQUE 和NOT NULL 约束） AUTOINCREMENT自动增长

说说主键：
主键用来唯一地标识某一条记录，可以是一个字段或多个字段(注：①)
主键设计原则：

主键应该是对用户没有意义的
永远也不要更新主键
主键不应包含动态变化的数据
主键应当由计算机自动生成
主键应该是INTEGER类型，这样的话我们也不同添加AUTOINCREMENT约束就自动增长了。

外键约束：利用外键约束来表示表与表之间的关系，一般情况是一个表的某个字段引用着另一张表的主键字段。

上面这条语句意义为创建一个t_student的表，字段有id，姓名，班级作为外键存在，此外键引用了t_class表的id字段作为值。

当两张表建立关系之后，想查询具体数据就需要使用多表连接查询。
进行表连接查询的时候只要分析一个等同的条件，并使用WHERE关键字进行条件查询，即可查询到想要的数据，比如t_class表和t_student表中，想查询学生，需要的条件就是学生的class = class的id。
另：多表连接嵌套查询的时候，可以使用AS关键字对表起别名来使语句更简洁易懂。

最后，其实这些SQL语句也不用必须背下来，懒人创造世界，我们也可以使用Nacicat，PLSQL等软件生成我们想要使用的语句，但是多记一些学一些东西总是好的，不是么？

注释①：主键如果是多个字段成为复合主键，需要注意的是复合主键并不是字段1作为主键的同时字段2也为主键，而是将字段1和字段2联合起来作为一个主键。
